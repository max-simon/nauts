# Operator Mode Test Environment

Operator mode uses per-account signing keys managed through NSC. This is recommended for production NATS deployments with operator/account hierarchy.

## Quick Start

```bash
# 1. Start NATS server
nats-server -c nats-server.conf

# 2. Start nauts auth callout service (in another terminal)
../../bin/nauts serve -c nauts.json

# 3. Test authentication (uses sentinel credentials + token)
nats --creds sentinel.creds --token '{"account":"APP","token":"alice:secret"}' sub "public.>"
nats --creds sentinel.creds --token '{"account":"APP","token":"bob:secret"}' pub public.test "Hello"
```

## Test Users

| User  | Password | Roles    | Permissions |
|-------|----------|----------|-------------|
| alice | secret   | readonly | Subscribe to `public.>` |
| bob   | secret   | full     | Pub/Sub to `public.>` |

## Files

| File | Description |
|------|-------------|
| `nauts.json` | nauts configuration |
| `nats-server.conf` | NATS server configuration (generated by NSC) |
| `auth.creds` | Credentials for callout service connection |
| `sk-AUTH.nk` | AUTH account signing key |
| `sk-APP.nk` | APP account signing key |
| `server-xkey.nk` | Curve key for request encryption |

## Configuration

```json
{
  "account": {
    "type": "operator",
    "operator": {
      "accounts": {
        "AUTH": {
          "publicKey": "<AUTH-account-public-key>",
          "signingKeyPath": "./sk-AUTH.nk"
        },
        "APP": {
          "publicKey": "<APP-account-public-key>",
          "signingKeyPath": "./sk-APP.nk"
        }
      }
    }
  },
  "policy": {
    "type": "file",
    "file": { "policiesPath": "../policies.json", "rolesPath": "../roles.json" }
  },
  "auth": {
    "file": [
      {
        "id": "local",
        "accounts": ["*", "AUTH"],
        "userPath": "../users.json"
      }
    ]
  },
  "server": {
    "natsUrl": "nats://localhost:4222",
    "natsCredentials": "./auth.creds",
    "xkeySeedFile": "./server-xkey.nk"
  }
}
```

## Setup from Scratch

### 1. Create Operator and Accounts

```bash
# Create operator
nsc add operator nauts-test --generate-signing-key --sys
nsc edit operator --require-signing-keys

# Create AUTH account with signing key
nsc add account --name AUTH
nsc edit account AUTH --sk generate

# Create APP account with signing key
nsc add account --name APP
nsc edit account APP --sk generate
```

### 2. Create Auth Service User

```bash
nsc add user --account AUTH --name auth
nsc generate creds --account AUTH --name auth > auth.creds
AUTH_USER_PUB=$(nsc describe user --account AUTH --name auth --json | jq -r .sub)
```

### 3. Generate Encryption Key

```bash
nk -gen curve > server-xkey.nk
XKEY_PUB=$(nk -pubout -inkey server-xkey.nk)
```

### 4. Enable Auth Callout

```bash
nsc edit authcallout \
  --account AUTH \
  --curve $XKEY_PUB \
  --auth-user $AUTH_USER_PUB \
  --allowed-account '*'
```

### 5. Generate Server Config

```bash
nsc generate config --mem-resolver --config-file nats-server.conf
```

### 6. Export Signing Keys

```bash
# Export AUTH signing key
nsc describe account AUTH --json | jq -r '.nats.signing_keys[0]' > sk-AUTH.pub
# Copy the corresponding .nk file from NSC store

# Export APP signing key
nsc describe account APP --json | jq -r '.nats.signing_keys[0]' > sk-APP.pub
# Copy the corresponding .nk file from NSC store
```

### 7. Update nauts.json

Set account public keys and signing key paths in the configuration.
